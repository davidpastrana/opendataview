Using your own Schema mapping to match any header field

(You can use any desired header pattern adding wildcard symbol '%' and use multiple csv-header tags (seperated by comma) to map them to the desired dictionary words - Dictionary words are defined below)



Dictionary words

id, name, csvname, latitude, longitude, email, username, website, source, otherinfo, description, date_published, date_updated, address, archive, city, country, currency, date, document, elevation, iconmarker, number, percentage, phone, population, postcode, private_mode, schedule, street, type, urlimage, year

Search via SQL Query language - Query data which was auto-detected or fields which you schema-mapped

(in front of the sql-query requires to write 'sql:', must have sql format and appear after the sql-clause 'where') - Dictionary words you can provide are below:



The following samples can be typed in into the search box (remember to always include 'sql:' name in front of the query, otherwise will make a standard search):
e.g. sql: latitude >= 48 and longitude <= 17
sql: latitude > 28 and longitude < -16
sql: lower(otherinfo) like '%citybikeogd%'
sql: lower(otherinfo) like '%fid:%spazierpunktogd%'
sql: (lower(otherinfo) like '%fid:%parkanlageogd%' or lower(otherinfo) like '%trinkbrunnen%1%')
sql: (lower(otherinfo) like '%fid:%parkanlageogd%' or lower(otherinfo) like '%hundezone%1%')
sql: (lower(otherinfo) like '%bezirk: 1 %' or lower(otherinfo) like '%zip: 1010%')
sql: csvname='city_walk.csv' and description <> ''
sql: csvname='city_walk.csv' and description like ''
sql: lower(name) like '%neue donau%'
sql: lower(description) like '%oper%'
sql: name like 'Polizei%'
sql: username = 'admin'
sql: id > 1 and id <= 5
sql: csvname = 'Police.csv'
sql: date_published = 'March 18 2019'
sql: date_updated = 'March 18 2019'
